### Phase 1: 数据库和核心服务

- [ ] **1.1 创建数据库迁移脚本**
  - 文件：`database/migrations/001_add_user_system.sql`
  - 创建 users 表（包含所有字段和索引）
  - 为 sites 表添加 user_id 外键
  - 插入默认 admin 账户（密码在安装时设置）
  - 验证：执行迁移脚本无错误

- [ ] **1.2 实现 UserService 基础类**
  - 文件：`src/Services/UserService.php`
  - 实现用户查询方法（getUserByUsername, getUserByEmail, getUserById）
  - 实现注册方法 register()（包含密码加密、唯一性校验）
  - 实现登录方法 login()（包含密码验证、Session 创建、登录锁检查）
  - 实现退出方法 logout()
  - 验证：单元测试覆盖所有方法

- [ ] **1.3 扩展 SettingsService**
  - 文件：`src/Services/SettingsService.php`
  - 添加 isRegistrationAllowed() 方法
  - 添加 toggleRegistration() 方法
  - 确保设置持久化到 admin.json 或 settings.json
  - 验证：开关状态可正确读取和保存

### Phase 2: 认证功能

- [ ] **2.1 创建注册页面**
  - 文件：`views/register.php`
  - HTML 表单：用户名、邮箱、密码、确认密码
  - 前端验证：必填字段、邮箱格式、密码长度、密码一致性
  - 显示错误提示区域
  - 验证：页面渲染正常，表单验证生效

- [ ] **2.2 实现注册 API**
  - 文件：`public/api/auth/register.php`
  - 接收 POST 数据（username, email, password）
  - 服务端验证（唯一性、强度）
  - 检查注册开关状态
  - 调用 UserService.register()
  - 返回 JSON 结果（成功/错误信息）
  - 验证：注册成功/失败场景均正确处理

- [ ] **2.3 改造登录页面**
  - 文件：`views/login.php`
  - 移除硬编码的 admin 检查
  - 支持任意注册账户登录
  - 添加"记住我"复选框（7 天自动登录）
  - 显示登录错误提示
  - 验证：现有 admin 账户和新注册账户均可登录

- [ ] **2.4 实现登录 API**
  - 文件：`public/api/auth/login.php`
  - 接收 POST 数据（username, password, remember）
  - 检查账户锁定状态
  - 验证密码
  - 创建 Session（2 小时或 7 天）
  - 记录登录失败次数
  - 更新最后登录信息
  - 返回 JSON 结果
  - 验证：登录成功创建 Session，失败记录次数

- [ ] **2.5 实现退出功能**
  - 文件：`public/api/auth/logout.php`
  - 销毁 Session
  - 清除 Cookies
  - 重定向到登录页面
  - 验证：退出后无法访问受保护页面

- [ ] **2.6 实现登录锁保护**
  - 在 UserService 中添加登录失败计数逻辑
  - 连续失败 5 次设置 locked_until 为 15 分钟后
  - 登录前检查 locked_until 是否过期
  - 验证：5 次失败后账户被锁定 15 分钟

## 验收标准

1. 用户可以自助注册账户（注册开关开启时）
2. 注册用户可以使用用户名密码登录
3. 登录用户可以管理自己的网站（添加、编辑、删除）
4. 登录用户可以查看自己网站的详细统计数据
5. 管理员可以在设置页面控制注册开关
6. 普通用户无法访问管理员功能
7. 所有 API 接口都有正确的权限验证
8. 密码、Session 等安全机制正常工作
9. 登录失败锁定机制正常工作
10. 单元测试和集成测试全部通过
